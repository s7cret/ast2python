# AST2Python documentation

Canonical documentation for the AST2Python 4.0.0 generator release line.

## Documents

| File | Purpose |
|---|---|
| `ARCHITECTURE.md` | Generator pipeline, modules, input validation, emitters, and ownership boundary. |
| `COMPATIBILITY.md` | Public contracts, supported lowering areas, visual-call policy, and non-goals. |
| `OPENPINE_PIPELINE.md` | How Pine2AST, AST2Python, PineLib, Backtest Engine, and OpenPine connect. |
| `DEVELOPMENT.md` | Local checks, AST input contract, visual policy, wheel smoke, and architecture budget. |
| `RELEASE_4_0.md` | 4.0.0 release summary, breaking changes, hardening notes, and checklist. |
| `SECURITY.md` | Safe code-generation model and integration restrictions. |

## Reading order

Start with `ARCHITECTURE.md`, then `COMPATIBILITY.md`, then `OPENPINE_PIPELINE.md`. Use `DEVELOPMENT.md` and `RELEASE_4_0.md` for release work.
